WHAT WAS BUILT
================================================================================

A production-grade Spring Boot microservice that:
1. Accepts CSV file uploads via REST API
2. Validates and parses CSV data
3. Persists data to PostgreSQL in a single atomic transaction
4. Returns structured JSON responses

REST Endpoint:
  POST /api/jobs/upload
  Accepts: multipart/form-data (CSV file)
  Response: HTTP 201 with JSON containing jobId, fileName, totalRecords, status

================================================================================
CORE COMPONENTS
================================================================================

1. REST Controller
   - FileUploadController: Handles POST /api/jobs/upload

2. Business Logic
   - FileUploadService: Orchestrates CSV upload workflow with @Transactional

3. Data Access
   - JobRepository: Spring Data JPA for Job entity
   - RecordRepository: Spring Data JPA for Record entity

4. Domain Models
   - Job: Represents file upload job (id, fileName, totalRecords, status, timestamps)
   - Record: Represents individual member record from CSV

5. CSV Processing
   - CsvParser: Parses CSV, validates columns, skips invalid rows
   - FileValidationUtil: File validation utilities

6. Error Handling
   - FileUploadException: Custom exception
   - GlobalExceptionHandler: @RestControllerAdvice for centralized error handling

7. Data Transfer Objects
   - FileUploadRequest: Wrapper for multipart file
   - FileUploadResponse: Response with jobId and metadata
   - MemberRecord: Internal DTO for CSV row data

================================================================================
DATABASE SCHEMA
================================================================================

Two tables auto-created by Hibernate:

JOBS TABLE:
  id              BIGINT PRIMARY KEY AUTO_INCREMENT
  fileName        VARCHAR(255) NOT NULL
  totalRecords    INTEGER NOT NULL
  status          VARCHAR(50) NOT NULL
  createdAt       TIMESTAMP NOT NULL (auto-populated)
  updatedAt       TIMESTAMP NOT NULL (auto-populated)

RECORDS TABLE:
  id                       BIGINT PRIMARY KEY AUTO_INCREMENT
  jobId                    BIGINT NOT NULL
  memberId                 VARCHAR(255) NOT NULL
  name                     VARCHAR(255) NOT NULL
  mobile                   VARCHAR(255) NOT NULL
  email                    VARCHAR(255) NOT NULL
  communicationPreference  VARCHAR(255) NOT NULL
  status                   VARCHAR(50) NOT NULL
  createdAt                TIMESTAMP NOT NULL (auto-populated)
  updatedAt                TIMESTAMP NOT NULL (auto-populated)

================================================================================
CSV FORMAT
================================================================================

Required header row with columns (case-sensitive):
  memberId,name,mobile,email,communicationPreference

Example data:
  M001,John Doe,9876543210,john@example.com,EMAIL
  M002,Jane Smith,9876543211,jane@example.com,SMS

Sample file: sample-members.csv (10 records included)

================================================================================
WORKFLOW
================================================================================

1. Client sends CSV file to POST /api/jobs/upload
2. Controller receives multipart file request
3. Service orchestrates processing:
   a. CsvParser validates and parses CSV
   b. Job record created with status="UPLOADED"
   c. Record entries created with status="RECEIVED"
   d. All DB operations in single @Transactional context
   e. Timestamps auto-populated by Hibernate
4. Controller returns HTTP 201 with response JSON
5. Response includes: jobId, fileName, totalRecords, status

================================================================================
ERROR HANDLING
================================================================================

HTTP 201 (Created): Successful upload
HTTP 400 (Bad Request): 
  - File is empty or null
  - File must be a CSV file
  - CSV file contains no valid records
  - Error parsing CSV file
HTTP 413 (Payload Too Large): File size exceeds maximum
HTTP 500 (Internal Server Error): Unexpected errors

All error responses follow standard format:
{
  "code": <http_status>,
  "message": "<error_message>",
  "timestamp": "<iso_timestamp>"
}
